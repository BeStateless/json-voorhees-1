item(int) \

An item list of types to also consider as an integer.

This mostly exists to help resolve the C-induced type ambiguity for the literal 0. It most prefers to be an int, but might also become a long or a pointer type.

#define JSONV_REGEX_INCLUDE   <regex>

Controls the regular expression engine to use.

By default, this will use the C++ Standard Library implementation. GCC versions below 4.8 will happy compile regular expressions, but will fail at runtime. If using GCC under 4.9, it is recommended that you set JSONV_REGEX_USE_BOOST to 1.
